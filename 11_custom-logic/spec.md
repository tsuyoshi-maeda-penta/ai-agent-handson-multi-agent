# AI Agent Design Specification: Refund Handling Agent System

## 1. 概要 (Overview)
本ドキュメントは、Antigravity CLI および Agent Development Kit (ADK) 2.0 を使用して開発する「返金対応 AI エージェント」のシステムデザイン仕様書である。

本エージェントシステムは、購入者からの返金要求に対して、ポリシーの検証、代替案の提示（クレジット付与）、および最終的な返答生成までを自動化する。多段階のサブエージェント構造を持ち、カスタムロジックパターン（シーケンシャルパターンとパラレルパターンの組み合わせ）を適用することで、堅牢かつ柔軟な自律処理を実現する。

## 2. システムアーキテクチャ (Architecture)
本システムは、全体を統括する「ルートエージェント」および「返答エージェント」を主軸とし、その配下に特定の役割を持ったサブエージェント群を配置する階層構造をとる。

### 2.1 制御フロー（カスタムロジック）
1. ユーザー対話と進捗チェック (ルートエージェント)
  お客様からのリクエストを受け付け、全体のワークフローを開始。最終的な対応完了・未完了のジャッジを行う。
2. 並列チェック (返金有効チェックエージェント / パラレルパターン)
  「購入者チェックエージェント」と「返金適格性チェックエージェント」を同時に（並列で）実行し、双方の結果を統合して返金の可否を判定する。
3. 条件分岐と逐次処理 (シーケンシャルパターン)
  * 返金可能の場合: 「返金処理エージェント」を呼び出し、実際の返金手続きを実行。
  * 返金不可の場合: 「クレジット付与エージェント」へ処理を移行。内部で「クレジット管理エージェント」による可否確認後、「クレジット付与決定エージェント」が額面を決定し付与を試みる。
4. 最終結果の集約 (返答エージェント)
  実行されたすべてのプロセスのコンテキストをまとめ、購入者に対して親切かつ明確な最終回答を生成する。

## 3. 技術仕様
* プログラミング言語は Python で実装します。
* Agent Development Kit (ADK) のバージョンは　2.x を使用する。ADK 1.x 系は互換性がないため使用しません。

### ADK のバージョンの指定方法
作成する Python プロジェクトの `pyproject.toml` には、ADK は次のバージョン指定を行います。

```
"google-adk[gcp]>=2.0.0,<3.0.0"
```

## 4. エージェント定義 (Agent Definitions)
### 4.1 ルートエージェント (Root Agent)
* 役割: お客様との対話窓口となり、プロセスの開始と最終的なステータス（完了/未完了）を監視・管理する。
* タイプ: 統括エージェント（シーケンシャル親）
* 入力: ユーザーからの問い合わせテキスト、購入履歴メタデータ
* 出力: 最終ステータス、各フェーズの実行ログ

### 4.2 返金有効チェックエージェント (Refund Validity Check Agent)
* 役割: サブエージェントを並列駆動し、返金処理を進めてよいか総合的に判定する。
* タイプ: パラレル・オーケストレーター
* サブエージェント:
  * 購入者チェックエージェント
  * 返金適格性チェックエージェント
* 判定ロジック: 両方のサブエージェントから True（適格）が返された場合のみ、返金ステータスを VALID とする。それ以外は INVALID とし、クレジット付与フローへ誘導する。

#### 4.2.1 購入者チェックエージェント (Buyer Verification Agent)
* 役割: 購入者が返金対応を受ける権利（会員ランク、過去の不正履歴、購入からの経過日数など）を保持しているか検証する。
* 入力: ユーザーID、注文ID
* 出力: 権利有無 (Boolean), 検証理由

#### 4.2.2 返金適格性チェックエージェント (Eligibility Check Agent)
* 役割: 申請された事象（商品の不具合、配送遅延など）が、返金ポリシーの対象に該当するかを検証する。
* 入力: ユーザーのリクエスト理由、商品カテゴリ、返品ポリシーデータ
* 出力: 適格性有無 (Boolean), 検証理由

### 4.3 返金処理エージェント (Refund Execution Agent)
* 役割: 返金有効チェックが通った案件に対し、決済システムと連携して実際の返金処理（トランザクションの取り消し等）を実行する。
* 入力: 注文ID、返金確定金額
* 出力: 決済処理結果（成功/失敗）、トランザクションID

### 4.4 クレジット付与エージェント (Credit Granting Agent)
* 役割: 返金不可と判定された場合、代替案としてストアクレジットの付与プロセスを逐次実行する。
* タイプ: シーケンシャル・オーケストレーター
* サブエージェント:
  * クレジット管理エージェント
  * クレジット付与決定エージェント

#### 4.4.1 クレジット管理エージェント (Credit Management Agent)
* 役割: 対象のユーザーがクレジット付与可能な状態か（上限に達していないか、アカウントが有効か）を確認する。
* 出力: 付与可否 (Boolean)

#### 4.4.2 クレジット付与決定エージェント (Credit Determination Agent)
* 役割: ユーザーの状況や不便の度合いに応じて、付与すべき最適なクレジット額（補填額）を算出し、付与を実行する。
* 出力: 付与されたクレジット額、付与完了ステータス

### 4.5 返答エージェント (Response Generation Agent)
* 役割: ルートエージェントおよび各サブエージェントが残した実行コンテキストを統合・解釈し、購入者へ提示する最終的な案内文（メールまたはチャットテキスト）を生成する。
* 入力: ワークフロー全体の実行履歴（返金成功、またはクレジット付与結果など）
* 出力: ユーザー向け最終案内テキスト（自然言語）
