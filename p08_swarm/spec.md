# AI Agent Design Specification: E-Commerce Autonomous Support Swarm

## 1. 概要 (Overview)
本ドキュメントは、Antigravity CLI を用いて開発・デプロイを行う、ECサイト向け高度カスタマーサポート＆トラブルシューティング AI エージェントシステムのデザイン仕様書である。
本システムは、ユーザーからの複雑かつ多岐にわたる問い合わせ（返品、注文変更、配送トラブル、クレームなど）に対して、人間の介在を最小限に抑え、自律的かつ柔軟に問題を解決することを目的とする。

従来のカスタマーサポートボットは、固定されたシナリオ分岐や単一の LLM プロンプトに依存しており、ユーザーが同時に複数の要望（例：「届いた商品が壊れていたので返品したい、ついでに別の注文の配送状況も教えてほしい」）を持っていた場合に柔軟な対応が困難であった。
本システムでは、マルチエージェント協調フレームワークである **Swarm パターン** を採用することで、各領域に特化した自律エージェントが動的にタスクを引き継ぎ（ハンドオーバー）、一気通貫で高度なトラブルシューティングを実行する。

## 2. システムアーキテクチャ (Architecture)
本システムは、1つの受付エージェント（Dispatcher）と4つの専門サブエージェントから構成される Swarm 型マルチエージェントアーキテクチャである。

### 2.1 トポロジーとインタラクションモデル
全てのやり取りは、まず受付エージェントがインターフェースとして集約するが、Swarm パターンの特性上、エージェント間で「コンテキスト」と「制御権」のハンドオーバーが双方向に発生する。

* **メッシュ型ハンドオーバー:** Dispatcher からサブエージェントへの処理委譲だけでなく、サブエージェントが対応中に別の要件が発生した場合（例：返品対応中に配送トラブルの確認が必要になった場合）は、サブエージェント間で直接、または Dispatcher を経由して制御権が自律的に移譲される。

### 2.2 システムコンポーネント図

```
    [ ユーザー (自然言語入力) ]
               │
               ▼
   ┌────────────────────────┐
   │   Dispatcher Agent     │◀───┐
   │ (受付・コンテキスト分析)   │    │
   └───────────┬────────────┘    │
               │                 │
   ┌───────────┼───────────┬─────┴─────┬───────────┐
   │           │           │           │           │
   ▼           ▼           ▼           ▼           ▼
┌───────────┐┌───────────┐┌───────────┐┌───────────┐┌───────────┐
│ 返金・返品  ││  注文変更  ││ 配送トラブル││  クレーム   ││ （相互に） │
│ 専門 Agent ││   Agent   ││ 専門 Agent││ 対応 Agent ││Handover可能│
└───────────┘└───────────┘└───────────┘└───────────┘└───────────┘
```

## 3. 技術仕様
* プログラミング言語は Python で実装します。
* Agent Development Kit (ADK) のバージョンは　2.x を使用する。ADK 1.x 系は互換性がないため使用しません。

### ADK のバージョンの指定方法
作成する Python プロジェクトの `pyproject.toml` には、ADK は次のバージョン指定を行います。

```
"google-adk[gcp]>=2.0.0,<3.0.0"
```

## 4. エージェント定義 (Agent Definitions)
### 4.1 受付エージェント (Dispatcher Agent)
* 役割: ユーザーからの最初の問い合わせ（自然言語）を受け付け、内容を分析して適切な専門エージェントに「ハンドオーバー（引継ぎ）」する。
* システムプロンプト:
  ```
  あなたはECサイトの総合受付エージェント（Dispatcher）です。
  ユーザーの最初の入力を注意深く分析し、ユーザーが何を求めているかを特定してください。
  自分で問題を解決しようとせず、速やかに適切な専門エージェントへ制御権を移譲（Handover）してください。
  複数の要望が含まれる場合は、最も優先度の高いもの、または時系列的に先に行うべきタスクのエージェントへ引き継ぎます。
  ```

### 4.2 返金・返品専門エージェント (Refund & Return Agent)
* 役割: 返金ポリシーの確認や、注文データベースの参照、返品手続きの受付を行う。
* システムプロンプト:
  ```
  あなたは返金・返品専門のエージェントです。
  注文データベースを参照して商品のステータスを確認し、社内の返金・返品ポリシー（例：購入後30日以内、未開封に限るなど）に合致しているかを厳密に判定します。
  条件を満たしている場合は、返品受付プロセスを自律的に進めてください。
  もしユーザーが配送遅延への不満を述べ始めた場合は、配送トラブル専門エージェントへハンドオーバーしてください。
  ```

### 4.3 注文変更エージェント (Order Modification Agent)
* 役割: 注文内容（配送先住所、商品の数量、カラー・サイズ、配送日時など）と変更内容を確認し、注文の変更・キャンセルを行う。
* システムプロンプト:
  ```
  あなたは注文変更専門のエージェントです。
  注文が「出荷準備完了」または「発送済み」ステータスになる前であるかを確認し、変更可能な範囲で注文内容の書き換えを行います。
  すでに発送済みの場合は変更ができない旨を丁寧に伝え、必要に応じて配送トラブル専門エージェント、または返品専門エージェントへ相談を促すかハンドオーバーしてください。
  ```

### 4.4 配送トラブル専門エージェント (Delivery Trouble Agent)
* 役割: 配送状況の追跡、配送キャリア（配送業者）への問い合わせステータスの確認、荷物紛失時の調査受付を行う。
* システムプロンプト:
  ```
  あなたは配送トラブル専門のエージェントです。
  配送追跡システムと連携し、ユーザーの荷物が現在どこにあるかを正確に把握・報告します。
  大幅な遅延や紛失の疑いがある場合、補償手続きや再送の手配を検討し、返金が必要な場合は返金・返品専門エージェントへコンテキストを引き継ぎます。
  ```

### 4.5 クレーム対応エージェント (Complaint Handling Agent)
* 役割: お客様のクレーム内容を親身にヒアリングし、感情分析（センチメント分析）を交えつつ、適切な謝罪や特別対応（クーポン発行、上位マネージャーへのエスカレーション）を行う。
* システムプロンプト設定:
  ```
  あなたはクレーム対応専門の特級エージェントです。
  最優先事項は、不満や怒りを感じているお客様の心情に寄り添い、丁寧かつ共感的な態度で対応することです。
  テクニカルな解決（返金や配送確認など）が必要な場合は、バックグラウンドで他のエージェントのツールや知見を借りるか、または合意の上でハンドオーバーを試みてください。
  必要に応じて、お詫びのクーポン発行や、人間のスーパーバイザーへのエスカレーション手続きを行います。
  ```
