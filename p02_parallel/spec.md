# AI Agent Design Specification: Trend Researcher

## 1. 概要 (Overview)
本ドキュメントは、Antigravity CLI を用いてビルドする、パラレルパターン（並列処理型）のマルチエージェントシステムの設計仕様書である。
本エージェントは、現在のトレンドを「国内メディア」「海外メディア」「SNS・世論」の3つの切り口から同時に調査し、最終的にそれらを1つの統合レポートとしてサマライズすることを目的とする。

## 2. システムアーキテクチャ (Architecture)
* **パターン名**: パラレルパターン (Parallel Orchestration Pattern)
* **構造**: 1つのオーケストレーター（Coordinator）が、3つの専門エージェントを並列に駆動し、各エージェントの出力結果を統合・集約する。

```
                    [ Trend Research Coordinator ]
                                  |
        +-------------------------+-------------------------+
        |                         |                         |
        v                         v                         v
[ Agent A: Domestic ]     [ Agent B: Global ]       [ Agent C: SNS/Public ]
        |                         |                         |
        +-------------------------+-------------------------+
                                  |
                                  v
                   [ Summary & Report Generation ]
```

## 3. 技術仕様
* プログラミング言語は Python で実装します。
* Agent Development Kit (ADK) のバージョンは　2.x を使用する。ADK 1.x 系は互換性がないため使用しません。

### ADK のバージョンの指定方法
作成する Python プロジェクトの `pyproject.toml` には、ADK は次のバージョン指定を行います。

```
"google-adk[gcp]>=2.0.0,<3.0.0"
```

## 4. エージェント定義 (Agent Definitions)

### 4.1. Coordinator Agent (オーケストレーター)
* **Role**: 全体の統括、並列プロセスの制御、最終サマライズの生成
* **System Prompt**:
    ```text
    ユーザーから与えられた調査テーマについて、Agent A、Agent B、Agent Cに対して同時に調査タスクを割り振ってください。
    すべてのエージェントから調査結果が返ってきたら、それぞれの視点（国内、海外、SNS）の差異や共通点を分析し、多角的な視点を含んだ構造的なトレンドレポートを作成してください。
    ```
* **Tool**: なし (LLM 推論のみ)

### 4.2. Agent A (国内メディア担当)
* **Role**: 国内の主要報道機関（新聞・TV等）のトレンドおよび論調の調査
* **System Prompt**:
    ```text
    あなたは国内メディア（大手新聞社、テレビ局など）の報道内容を調査する専門エージェントです。
    指定されたテーマについて、日本のマスメディアがどのような文脈で報じているか、何が注目されているかを客観的に抽出・分析してください。
    ```
* **Tool**: Google Search Tool

### 4.3. Agent B (海外メディア担当)
* **Role**: 海外の主要ニュースソース（ロイター、BBC等）のトレンドおよび論調の調査
* **System Prompt**:
    ```text
    あなたは海外メディア（ロイター、BBC、CNNなど）の報道内容を調査する専門エージェントです。
    指定されたテーマについて、国際社会や主要な海外メディアがどのような視点から報じているか、日本国内の論調との違いを含めて抽出・分析してください。
    ```
* **Tool**: Google Search Tool

### 4.4. Agent C (SNS・世論担当)
* **Role**: SNS（Xなど）におけるリアルタイムなトレンドと世論の調査
* **System Prompt**:
    ```text
    あなたはSNS（Xなど）のリアルタイムなトレンドや世論を調査する専門エージェントです。
    指定されたテーマについて、一般ユーザーの間でどのようなワードがバズっているか、どのような感情（ポジティブ・ネガティブ等）や意見が多く見られるかをリアルタイムな視点で抽出・分析してください。
    ```
* **Tool**: Google Search Tool

## 5. ワークフロー・データフロー (Workflow & Dataflow)

### Step 1: 入力 (Input)
* ユーザーが調査したい「テーマ/キーワード」を入力する。

### Step 2: ファンアウト (Fan-Out / Parallel Execution)
* Coordinator は入力を受け取り、以下の 3 つのタスクを**同時に（並列で）**発行する。
    1.  `Agent A` へ国内メディアの調査を依頼
    2.  `Agent B` へ海外メディアの調査を依頼
    3.  `Agent C` へSNS・世論の調査を依頼

### Step 3: ファンイン (Fan-In / Aggregation)
* Coordinator は、3 つのエージェントすべてが処理を完了するのを待機する。
* 各エージェントからのペイロード（テキストデータ）を収集する。

### Step 4: 出力 (Output)
* Coordinator が収集した 3 種類のデータを元に、以下の構成で最終レポートを出力する。
    * **総合サマリー** (全体の要約)
    * **国内メディアの視点** (Agent A の結果ベース)
    * **海外メディアの視点** (Agent B の結果ベース)
    * **SNS・世論の動向** (Agent C の結果ベース)
    * **考察・比較分析** (メディア間でのギャップや特徴のまとめ)
